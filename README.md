# AIsha Podpiski — Трекинг конверсий подписок

Telegram-бот + веб-сервер для отслеживания конверсий подписок на каналы (Telegram и MAX) с отправкой данных в Яндекс Метрику.

## Как работает

1. На лендинге размещается JS-скрипт, который считывает ClientID из куки Яндекс Метрики
2. При клике на кнопку «Подписаться» происходит переход на `/go` с ClientID
3. **Telegram**: сервер создаёт одноразовую invite-ссылку, пользователь подписывается, бот фиксирует подписку
4. **MAX**: сервер редиректит на канал, бот обнаруживает нового подписчика через MAX API
5. Через 7 минут бот проверяет, что подписка сохранилась, и отправляет конверсию в Метрику

## Установка

```bash
git clone https://github.com/dmgubanov/AIsha-Podpiski.git
cd AIsha-Podpiski
pip install -r requirements.txt
cp .env.example .env
# Отредактируйте .env — заполните TELEGRAM_BOT_TOKEN, ADMIN_ID, TRACKING_BASE_URL
```

## Запуск

```bash
python src/main.py
```

## Настройка через бота

1. Отправьте `/start` боту
2. Добавьте канал (Telegram или MAX)
3. Укажите счётчик и токен Яндекс Метрики
4. Скопируйте JS-сниппет из инструкции на лендинг

## Переменные окружения

| Переменная | Обязательная | Описание |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | Да | Токен Telegram-бота |
| `ADMIN_ID` | Да | ID администратора (через запятую для нескольких) |
| `ACCESS_TOKEN_ENCRYPTION_KEY` | Рекомендуется | Ключ шифрования токенов (32+ символов) |
| `TRACKING_BASE_URL` | Да | Публичный URL сервера (например, `https://go.example.com`) |
| `TRACKING_WEB_PORT` | Нет | Порт веб-сервера (по умолчанию 8080) |
| `MAX_BOT_TOKEN` | Нет | Токен MAX-бота (для трекинга MAX) |
| `MAX_AUTO_CONNECT_ENABLED` | Нет | Включить MAX polling (`true`/`false`) |

Полный список — в `.env.example`.
